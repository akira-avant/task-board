# PJ Tools — デザインコンセプト & デザインシステム

> **プロジェクト管理 Web アプリケーション**
> "for builders who ship" — 現場で実際にプロジェクトを回す人のためのツール

---

## 1. デザインコンセプト

### 1.1 コアテーマ: **Workshop / Craft（工房・手仕事）**

PJ Tools は、プロジェクトを「職人が道具を使って構築する行為」として捉えます。
灯台 × ハンマーのロゴが象徴する通り、**進路を照らす（ディレクション）** と **組み上げる（クラフト）** の両方を担う道具です。

### 1.2 三つの設計原則

| 原則 | 意図 |
|---|---|
| **Paper, not pixel**（紙のように） | モニタ上でも "紙の帳簿" のような落ち着きと可読性。白ではなく淡黄色 (`#faf7f0`) をベースに、印字感のある墨インクで情報を並べる。 |
| **Hand, not machine**（手の跡を残す） | 見出しに手書き風ディスプレイ体、サブラベルに手書きセリフを使う。完全に整った "システム" ではなく "帳面" を感じさせる。 |
| **Burgundy as authority**（権威としてのバーガンディ） | アクセントは深いワインレッド `#3a1e24`。彩度の高い赤ではなく、古い図書館や事務局の背表紙を思わせる色で "長く使う道具" の質感を付与する。 |

### 1.3 ターゲットユーザー

- **PM / PL**: 顧客とチームの間に立つプロジェクトマネージャー
- **メンバー**: タスクを受け取って動くチームメンバー
- **PMO / 経営**: 複数案件を横断して状況を把握する層

3層それぞれが同じツールを使いながら、必要な粒度の情報（俯瞰 / 担当範囲 / タスク詳細）を無理なく行き来できることを目指します。

### 1.4 AI スロップを避ける方針

- グラデーション背景・装飾アイコン・絵文字は使用しない
- 角丸＋左ボーダー＋淡色カードの "LP 風" レイアウトを避ける
- 統計・装飾的な数字バッジを並べない（"Data slop" 禁止）
- すべての要素は "情報として読まれる" ために存在する

## 2. カラーシステム

### 2.1 ベースパレット — Paper & Ink

| トークン | HEX | 用途 |
|---|---|---|
| `--paper` | `#faf7f0` | ベース背景（本文領域） |
| `--paper-2` | `#f4efe4` | やや深い紙色（zebra / hover） |
| `--paper-3` | `#ece5d3` | 更に深い段（テーブルヘッダ等） |
| `--paper-edge` | `#e7dfcc` | ボーダー、区切り線 |
| `--ink` | `#1a1410` | 本文文字色（真っ黒ではなく墨寄り） |
| `--ink-2` | `#3a312a` | 強調テキスト |
| `--ink-3` | `#5c5047` | セカンダリテキスト |
| `--ink-4` | `#857767` | ラベル・補助情報 |
| `--ink-5` | `#aea294` | disabled / plate 罫線 |

### 2.2 ブランドアクセント — Burgundy

| トークン | HEX | 用途 |
|---|---|---|
| `--burgundy` | `#7a2e3a` | プライマリボタン、強調アクセント |
| `--burgundy-2` | `#3a1e24` | サイドバー背景（採用色） |
| `--burgundy-tint` | `#f5e6e3` | Hover / 選択行 |
| `--burgundy-ink` | `#f2e4e0` | ダーク背景上のテキスト色 |

サイドバーの背景に `--burgundy-2` を採用。重く・落ち着いた権威を演出しつつ、"道具" としての信頼感を出す。

### 2.3 セマンティック

| トークン | 文字色 | 背景色 | 意味 |
|---|---|---|---|
| `--ok` | `#2f6b3a` | `#e7f0e7` | 完了 / 成功 |
| `--info` | `#1e5a96` | `#e5eef9` | 進行中 |
| `--warn` | `#a0541a` | `#fbe8d8` | 保留 |
| `--risk` | `#a01a28` | `#fbeaea` | 要注意 / 遅延 |
| `--red` | `#c94a2e` | — | 日付変更など変化の印 |

## 3. タイポグラフィ

### 3.1 フォントスタック

| 役割 | トークン | フォント |
|---|---|---|
| ディスプレイ（見出し） | `--font-display` | `Caveat Brush`, `Shadows Into Light Two`, cursive |
| ハンド（ラベル） | `--font-hand` | `Shadows Into Light`, `Caveat`, cursive |
| 本文 | `--font-body` | `Inter`, `Hiragino Kaku Gothic ProN`, `Yu Gothic UI`, `Meiryo`, sans-serif |
| モノスペース | `--font-mono` | `JetBrains Mono`, `SF Mono`, Menlo, monospace |

**使い分けの原則:**
- **ディスプレイ**: プロジェクト名 / セクション H3 / ブランド名 — "人の気配" を出したい場所のみ
- **ハンド**: K/V のキー、eyebrow、ナビ、タグ、補助文 — "メモ書き" の質感
- **本文**: テーブルデータ、入力値、段落 — 徹底して読みやすさ優先
- **モノ**: 列キー名、HEX 値、コード

### 3.2 スケール

| サイズ | 用途 |
|---|---|
| 44px / display | プロジェクト名 (`h1`) |
| 28px / display | ブランドロックアップの "PJ Tools" |
| 22px / display | セクション見出し (`h3`) |
| 20px / display | インスペクタ見出し |
| 16px / display | サブセクション見出し |
| 14px / body | 本文・ナビ |
| 13px / body | テーブルセル、入力 |
| 12.5px / body | 補助文、hint |
| 12px / hand (大文字 + tracking 0.08–0.16em) | eyebrow、K/V のキー |
| 11–11.5px / mono | 列キー、HEX |

## 4. レイアウト・グリッド

### 4.1 アプリシェル

```
+----+---------------------------------------------------+
| S  | Topbar (brand + nav + search + icon buttons)      |
| i  +---------------------------------------------------+
| d  |                                                   |
| e  |  Page frame (max 1600px, padding 24px 32px)       |
| b  |                                                   |
| a  |   - Breadcrumb                                    |
| r  |   - ProjectHeader                                 |
|    |   - Tabs                                          |
|    |   - Section cards                                 |
+----+---------------------------------------------------+
```

- **Sidebar**: 76px 固定幅、`position: sticky`。Burgundy `#3a1e24`。
- **Topbar**: 54px 高、白背景、下ボーダー `--paper-edge`。
- **Page frame**: 最大幅 1600px、両サイド 32px の余白。
- **Section 間の余白**: 24px。

### 4.2 ラディウス

| トークン | px | 用途 |
|---|---|---|
| `--r-sm` | 4 | チップ、極小要素 |
| `--r-md` | 6 | ボタン、入力 |
| `--r-lg` | 10 | カード、モーダル |

### 4.3 シャドウ

| トークン | 値 | 用途 |
|---|---|---|
| `--shadow-1` | 軽い内陰 | カード、テーブルヘッダ |
| `--shadow-2` | 通常 | ホバー、ドロップダウン |
| `--shadow-pop` | 強め | モーダル、インスペクタ |

一方で、"紙" を表現するために **`2px 2px 0 var(--ink)`** のハードシャドウも使用（ボタンや小カードの一部）。これは手書きスタンプ風の印影。

## 5. コンポーネント

### 5.1 ブランドロックアップ

```
[灯台アイコン]  PJ Tools
               FOR BUILDERS WHO SHIP
```

- アイコン: `MiniMark` コンポーネント（灯台 + ハンマーTの手描きSVG、feTurbulence で荒らし処理）
- 名称: `--font-display`, 28px, わずかに -1deg 回転
- タグライン: `--font-hand`, 11px, uppercase, letter-spacing 0.14em

### 5.2 サイドバー

- 幅 76px、Burgundy `#3a1e24` 背景
- ロゴ: 白紙の上にアイコン、-2deg 回転（紙を貼った風）
- ナビアイテム: アイコン + 日本語 1-2 文字ラベル（60x60）
- アクティブ: 左側に 3px 幅の `#d8a0a8` バー

### 5.3 セクションカード

```
┌─────────────────────────────────────────────┐
│ [1] セクションタイトル   補足説明  [操作]   │
├─────────────────────────────────────────────┤
│                                             │
│   本体                                       │
│                                             │
└─────────────────────────────────────────────┘
```

- ヘッダ左に丸番号（Burgundy 背景 + 白文字）
- タイトルは `--font-display`, 22px
- ヒントは `--font-hand`, 12.5px, `--ink-3`

### 5.4 ステータスチップ

丸ドット + ラベル、pill 形、手書き体:

- `chip-todo`: グレー / 紙色
- `chip-wip`: 青 / 水色
- `chip-done`: 緑 / 薄緑
- `chip-risk`: 赤 / 薄赤
- `chip-hold`: 橙 / 薄橙

### 5.5 テーブル（WBS）

- ヘッダ: `--paper-3` 背景、`--font-hand`、uppercase、tracking 0.06em
- 行: 12.5px 本文、垂直中央揃え
- Hover: `--burgundy-tint` で行全体を淡く染める
- 日付変更セル: 赤 + `※` 接頭
- ステップ見出し行: `--ink` ベタ背景 + 紙色文字（章扉のような断ち）

### 5.6 ボタン

| クラス | 背景 | 文字色 | 用途 |
|---|---|---|---|
| `btn-primary` | `--burgundy` | white | 主要アクション |
| `btn` | `--paper` | `--ink` | セカンダリ、hard shadow 付き |
| `btn-ghost` | transparent | `--ink-3` | 補助、filter 切替など |

### 5.7 インスペクタ（右ドロワー）

- 幅 420px、右からスライドイン
- ヘッダに WBS コード（mono）+ タスク名（display）
- K/V グリッド（キー = 手書き uppercase / 値 = 本文）

### 5.8 フォーム

- 入力: 白背景 + `--paper-edge` ボーダー、フォーカス時 Burgundy リング（`0 0 0 3px rgba(122,46,58,0.12)`）
- Checkbox: 手書き風カスタム（塗りつぶしで on）

---

## 6. アイコノグラフィ

- 線画アイコン、stroke-width 1.6, `stroke-linecap: round`, `stroke-linejoin: round`
- サイズは 20–22px で統一
- 色は `currentColor` を使い、親要素のテキスト色を継承
- 装飾的な絵文字や 3D アイコンは**使わない**

---

## 7. インタラクション

| 種類 | 基本値 |
|---|---|
| トランジション | `120ms` ease（色・背景） |
| ホバー | 薄い紙色 or Burgundy tint |
| 選択行 | `--burgundy-tint` + 左 3px バー |
| フォーカスリング | Burgundy, 3px, alpha 0.12 |

長い・派手なアニメーションは避け、"手で触れた" 感覚だけを残す。

## 8. 情報設計

### 8.1 画面階層

```
案件管理 (deals)
  └── プロジェクト一覧
       └── プロジェクト詳細 (4 tabs)
            ├── 基本情報
            ├── スケジュール (XLSX 取込 / 列設定 / WBS)
            ├── 週次進捗報告
            └── PJ スケジュール (ガント)
チーム管理 (teams)
顧客管理 (customers)
ユーザー管理 (users)
設定 (settings)
```

### 8.2 プロジェクト詳細の主要タスク

1. **XLSX 取込**: チームスケジュール (.xlsx) をアップロード → 自動でステップに反映
2. **列設定**: 進捗報告 PPT に出力する列の表示/非表示、順序、ラベル変更
3. **ステップ管理**: WBS 一覧、ステータス、日付、担当チーム/会社の可視化
4. **PPT 出力**: ステップ単位のスタイル設定

## 9. ブランドボイス

- **敬体ベース、ただし簡潔に**: 「〜します」「〜されます」
- **専門用語はそのまま**: WBS, PJ, PM, PMO, PPT 等は現場の語彙を尊重
- **カタカナ英語は必要最小限**: 「エクスポート」○ 「ダウンロード」○ ただし「レジスター」✗
- **数値 + 単位は半角**: 12名、42%、2026/04/18
- **操作動詞は簡潔**: 保存 / 削除 / 追加 / 編集 / 反映

---

## 10. アクセシビリティ

- 本文コントラスト比 4.5:1 以上（`--ink` on `--paper` = 14.1:1）
- Burgundy ボタンの文字色は常に白で 5.3:1 以上確保
- フォーカスリングを常に表示（`outline: none` で削除した箇所は別途 ring を用意）
- アイコン単独のボタンには必ず `title` と `aria-label`
- 手書き体フォントは**見出し・ラベルのみ**に限定し、本文・データには使わない

## 12. 将来拡張メモ

- **ダークモード**: サイドバー (Burgundy-2) を基準に反転。ベースを `--ink` に、インクを `--paper` に。
- **印刷スタイル**: 紙色をオフして真っ白、色を黒に落とす。PPT 出力の視認性と揃える。
- **モバイル**: 現状はデスクトップ前提。モバイルはサイドバーをボトムタブに変換し、プロジェクト詳細はタブ → アコーディオンに折る。
